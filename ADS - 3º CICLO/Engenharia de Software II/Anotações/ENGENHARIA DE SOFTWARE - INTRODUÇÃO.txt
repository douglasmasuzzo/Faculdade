** PRINCÍPIOS OPERACIONAIS

   - 1º PRINCÍPIO
    + o domínio da informação de um problema precisa ser representado e entendido

   - 2º PRINCÍPIO
    + as funções a serem desenvolvidas pelo software devem ser definidas

   - 3º PRINCÍPIO
    + o comportamento do software 

   - 4º PRINCÍPIO
    + os modelos mostram informações, funções e comportamentos devem ser particionados de um modo que revele detalhes em forma de camadas ( hierarquia )

   - 5º PRINCÍPIO
    + a tarefa de análise deve ir da informação até os detalhes de implementação

 * O MODELO DEVE FOCAR EM REQUISITOS QUE SÃO VISÍVEIS NO PROBLEMA OU DOMÍNIO DO NEGÓCIO. O NÍVEL DE ABSTRAÇÃO DEVE SER RELATIVAMENTE ALTO 

 * CADA ELEMENTO DO MODLELO DE ANÁLISE DEVE CONTRIBUIT PARA UM ENTENDIMENTO GLOBAL DOS REQUISITOS DE SOFTWARE E FORNECER UMA VISÃO DE DOMÍNIO DA INFORMAÇÃO / FUNÇÃO / COMPORTAMENTO DO SISTEMA
