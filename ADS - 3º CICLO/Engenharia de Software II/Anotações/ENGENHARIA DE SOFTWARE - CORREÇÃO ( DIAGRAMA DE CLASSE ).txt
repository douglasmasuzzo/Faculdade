** ANOTÇÕES

 -> COMENTÁRIO : RELACIONAMENTO ENTRE AS CLASSES DE PEDIDO E ITEM_PEDIDO
  + PEDIDO -> ITEM_PEDIDO ( 1...* ||  * ) 

 -> COMENTÁRIO : A CLASSE "CARINHO" TAMBÉM PODE SER INCLUIDA COMO PEDIDO

 -> COMENTÁRIO : EM RELAÇÃO AO PAGAMENTO, A CLASSE DEVE OBTER O CÓDIGO DO PAGAMENTO

 -> COMENTÁRIO : NÃO É ADEQUADO INCLUIR MÉTODOS DE OUTRAS CLASSES EM UMA CLASSE DIFERENTE 

 -> COMENTÁRIO : NÃO DEFINIR UM ATO DE AÇÃO COMO NOMENCLATURA DE UM MÉTODO ( CADASTRAR -> REPRESENTAR AÇÃO DO CRUD ) 

 -> COMENTÁRIO : DEFINIR QUAIS SÃO OS TIPOS DE AÇÕES DE ACORDO COM O OBJETO ( codigo.Aluno | nome.Aluno | historico.Aluno )

 -> COMENTÁRIO : SOBRE O ENDEREÇO DE UM CLIENTE, É IMPORTANTE CONSIDERAR O LOGRADOURO / COMPLEMENTO E OUTRAS QUESTÕES RELACIONADA AO NOSSO SISTEMA ( ENTREGA ) 

 -> CORREÇÃO : TORNAR ALGUNS ATRIBUTOS PÚBLICOS ( # ) NO NOSSO DIAGRAMA

 -> CORREÇÃO : INSERIR A CLASSE DE PAGAMENTO E DEFINIR OS PROCESSOS DE PAGAMENTO 

 -> CORREÇÃO : REVISAR NOMENCLATURAS E REVISAR OS DEMAIS CONCEITOS DO DIAGRAMA DE CLASSE

 -> CORREÇÃO : SE ADICIONAR A CLASSE RELATÓRIO AO NOSSO SISTEMA, A CLASSE NÃO SERÁ "ARMAZENADA" ( SENDO DESENVOLVIDA APENAS NA CODIFICAÇÃO )
 

 -> INSTRUÇÕES DE CRIAÇÃO DE UM DIAGRAMA DE CLASSE

  + IDENTIFICAR OBJETOS
  + CRIAR CLASSES
  + DEFINIR ATRIBUTOS DIRECIONADOS A CLASSE DESENVOLVIDA
  + CRIAR MÉTODOS DA CLASSE QUE CONTROLA ESTE OBJETO
  + ESPECIFICAR AS AÇÕES DESENVOLVIDAS PELO CRUD
  + SEGUIR UMA NOMENCLATURA DE NOMES DE CLASSES ( segundo a padrão nominativo )
  + DETERMINAR O TIPO DE ACESSO DESTA CLASSE ( MAJORITARIAMENTE COMO TIPO PÚBLICO )
  + ESTABELECER OS RELACIONAMENTOS ENTRE AS CLASSES ( MULTIPLICIDADE )
  + AÇÕES RECOMENDADAS PARA SEREM APLICADAS, CONFORME O SISTEMA
