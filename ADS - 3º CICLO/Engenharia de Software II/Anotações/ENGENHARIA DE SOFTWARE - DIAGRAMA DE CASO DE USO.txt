** DIAGRAMA DE CASO DE USO - NOSSO PROGRAMA 

   ( ATOR : CLIENTE ) 

   
    - BASE CASO DE USO <- EXTEND 
	+ A SETA SEGUE O FLUXO PARA O FLUXO BASE 

    - BASE CASO DE USO -> INCLUDE
        + A SETA SEGUE O FLUXO PARA A CLASSE 

   -- CORREÇÃO DA DEFINIÇÃO " IMPLEMENTS " SUBSTITUINDO PARA " INCLUDE " NO CADASTRO DE USUÁRIO

   -- REDEFINIR A DESCRIÇÃO DOS REQUISITOS ( RF09 | RF02 )

   -- CORRIGIR A ÁREA DE " GERENCIAMENTO DE ENTREGA DO PRODUTO " PARA " EXTEND "

   -- CORRIGIR A ÁREA RELACIONADA A " RASTREABILIDADE DE PEDIDO " PARA " RASTREABILIDADE DE ENTREGA "

   -- CORRIGIR A DESCRIÇÃO DO CASO DE USO DE " IMPLEMENTS " PARA " EXTENDS ".

   -- CODIFICAR A ÁREA RELACIONADA A " COMPRA ONLINE " | " GERENCIAMENTO DE ENTREGA DO PRODUTO "
